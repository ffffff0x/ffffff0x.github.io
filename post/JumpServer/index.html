JumpServer利用

JumpServer，安装基数最大的开源堡垒机，已成为堡垒机的代名词也是攻击中的必争之地。

JumpServer

在攻击场景中，JumpServer是攻击者的必争之地。如果获得堡垒机权限，就可获得进入内网甚至是直接管理堡垒机上所有的资产.

漏洞利用

无凭证

WS未授权访问

由于JumpServer程序中连接websocket的接口未做授权限制，导致攻击者可构造恶意请求获取服务器敏感信息，通过敏感信息中的相关参数，可构造请求获取相应token，进而可通过相关API操作来执行任意命令

影响版本：

  • JumpServer < v2.6.2
  • JumpServer < v2.5.4
  • JumpServer < v2.4.5
  • JumpServer = v1.5.9
  • blackjump

CVE-2023-42442

因为/api/v1/terminal/sessions/接口可以匿名访问导致，最终可以实现未授权下载会话回放，访问该接口返回200，基本就存在漏洞。

影响版本：

  • 3.0.0 <= JumpServer <= 3.5.4

  • 3.6.0 <= JumpServer <= 3.6.3

  • blackjump

下载出来的会话回放可以放进VideoPlayers进行播放

CVE-2023-42820

JumpServer伪随机数导致的未授权任意用户密码重置

影响版本:

  • v2.24 - v3.6.4
  • blackjump

CVE-2023-43650

重置密码Token可爆破漏洞
